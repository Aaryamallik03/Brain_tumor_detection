3. Model Architecture

Backbone: ResNet-18 (Fast/Deployed) / ResNet-50 (Full Training)

ResNet (Residual Network) was introduced by Microsoft Research in 2015. Its key innovation is the residual (skip) connection — a shortcut that lets gradients flow directly through layers during backpropagation, solving the vanishing gradient problem in very deep networks.

Input → [Conv Block] ──→ [+ skip] → [Conv Block] ──→ [+ skip] → ... → FC Layer → Output
               ↑_________________________|       ↑_________________________|
                    Residual Connection               Residual Connection

ResNet-18 vs ResNet-50

PropertyResNet-18ResNet-50

FC Input Size5122048
SpeedFast — CPU friendlySlower — GPU preferred
Used in this projectDefault deployed modelFull training mode

How ResNet is Used Here

Step 1 — Load pretrained weights: The backbone is initialized with ImageNet weights (trained on 1.2 million images, 1,000 classes).

Step 2 — Replace the final layer: The original 1,000-class output head is swapped for a new 4-class layer:

in_features = model.fc.in_features   # 512 (ResNet-18) or 2048 (ResNet-50)
model.fc = nn.Linear(in_features, 4) # 4 output neurons = 4 tumor classes

Step 3 — Fine-tune end-to-end: All layers are trained on the MRI dataset. The network adapts its feature detectors from general image patterns to MRI-specific anatomy.

Step 4 — Auto-detect at inference: The app inspects the fc.weight shape from the saved .pth file — if it's [4, 512] it loads ResNet-18; if [4, 2048] it loads ResNet-50. No manual config needed.

ImageNet Normalization Values

Mean : [0.485, 0.456, 0.406]   # R, G, B channels
Std  : [0.229, 0.224, 0.225]   # R, G, B channels

Why ImageNet normalization? The backbone was pretrained using these statistics. Using the same values at fine-tuning and inference ensures activations are in the same numerical range the network learned from, improving convergence and accuracy.

5. Data Augmentation

Applied only during training, not at inference. Artificially increases dataset diversity to reduce overfitting.

TechniqueParametersPurpose
Random Horizontal Flipp = 0.5Brain anatomy is roughly symmetric; flip doubles effective data
Random Rotation±10 degreesSimulates slight head tilt variation in real scans
transforms.RandomHorizontalFlip()
transforms.RandomRotation(10)

Why no color jitter or brightness shifts? MRI pixel intensity carries diagnostic meaning. Aggressive color augmentation would distort clinically relevant features and corrupt the training signal.

Optimizer — Adam

Adam (Adaptive Moment Estimation) adapts the learning rate per parameter using first and second moment estimates. Well-suited for transfer learning because different layers may need different effective update magnitudes.

Scheduler — ReduceLROnPlateau

Monitors validation accuracy after each epoch. If accuracy does not improve for 2 consecutive epochs, the learning rate is halved. Prevents overshooting optimal weights late in training.

Loss — CrossEntropyLoss

Standard multi-class classification loss combining log-softmax and negative log-likelihood. Penalizes confident wrong predictions more heavily than uncertain ones.

7. Inference Pipeline

When a user uploads an MRI scan:

StepAction
1Image loaded and converted to RGB
2Preprocessing applied: resize → tensor → normalize
3Forward pass through ResNet → 4 logits produced
4Softmax converts logits to probabilities (sum = 100%)
5Argmax selects predicted class
6Confidence reported as highest probability %
7Gradient saliency computed via backward pass
8Three visualizations generated (heatmap, orange spot, annotation)
9Result stored in SQLite monitoring database

8. Visualization Methods

Gradient Saliency — Red Heatmap

A backpropagation-based method:

heat = |∂(top class score) / ∂(input pixel)|  averaged over color channels, normalized to [0,1]

High gradient pixels = regions that most influenced the prediction. Overlaid as a red tint on the original MRI.

Orange Tumor Spot Overlay

Built on the same gradient map with a warm colormap:

Bounding Box Annotation

Saliency mask thresholded at 0.55 → bounding box fitted to active rows/columns → dashed red box with L-shaped corner ticks, red banner "TUMOR REGION DETECTED", and coordinate readout.

10. Limitations

  • Trained on a curated Kaggle dataset — real-world MRI scans vary in resolution, contrast, and acquisition protocol
  • Gradient saliency is an approximation — shows where the model is sensitive, not a true anatomical segmentation
  • No clinical validation has been performed
  • For educational and research use only — not a medical device
